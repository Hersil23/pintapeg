# Reglas de trabajo para este proyecto

## Autonomia
- Ejecuta todos los cambios de HTML, CSS y JS sin pedir confirmacion previa.
- Solo pregunta antes de actuar si el cambio involucra: base de datos, credenciales, eliminacion de archivos, cambios en .htaccess, o modificaciones al backend PHP que afecten logica de negocio.
- No pidas confirmacion para cambios de estilos, estructura HTML, partials PHP, o assets.

## Flujo de trabajo
- Aplica los cambios directamente y reporta que archivos modificaste al final.
- Si algo tiene mas de una forma correcta de hacerse, elige la mejor opcion tecnica y explica brevemente por que la elegiste.
- No hagas preguntas de diseno, esas decisiones vienen del prompt.
